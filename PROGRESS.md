# 📈 CareSight-Agent 项目进度里程碑 (Roadmap & Progress)

本项目旨在为视障人士提供一个从感官到逻辑的闭环医疗助手。以下是目前的开发进度与未来规划。

---

## 🟢 第一阶段：核心架构与知识对齐 (已完成)
- [x] **基础设施搭建**: 确定 LangGraph 状态机架构，配置 Gemini API 与本地 Ollama 环境。
- [x] **数据路径规范化**: 确立 `data/` 目录规范，将持久化向量库 `chroma_db` 迁移至 `data/` 下。
- [x] **RAG 引擎启动**: 成功对接 Huatuo-26M 医疗数据集，实现基于语义相似度的专业医疗检索。

## 🟢 第二阶段：无障碍语音交互集成 (已完成)
- [x] **ASR 语音识别**: 集成 `faster-whisper`。为了适配 Windows 安全策略，采用 C++ 引擎绕过 Numba DLL 限制。
- [x] **TTS 语音合成**: 接入 `edge-tts` (微软神经语音)，提供自然的人类声音反馈。
- [x] **多模态链路打通**: 支持在终端通过“回车”切换文字/语音录制，并实现 AI 回复的自动语音播报。
- [x] **输出文本清洗**: 实现正则过滤，防止 TTS 读出 Markdown 特殊符号（如 `*`, `#`）。

## 🟡 第三阶段：交互体验优化与鲁棒性增强 (进行中)
- [ ] **ASR 鲁棒性**: 处理用户长时间录音或背景噪音的情况。
- [ ] **多轮对话上下文管理**: 优化 LangGraph 状态保存，确保 AI 记忆之前的问诊信息。
- [ ] **错误处理增强**: 针对 API 超时、网络波动提供语音级别的错误提示。

## 🔴 第四阶段：视觉感知与 OCR 模块 (待启动)
- [ ] **药品识别**: 引入视觉模型解析药盒、说明书。
- [ ] **化验单分析**: 使用 OCR 提取关键医学指标并结合 RAG 进行白话文解读。
- [ ] **环境导向**: 实时感知拍摄画面质量，提供语音快门引导。

## 📅 更新日志 (Recent Updates)
- **2026-04-03**: 迁移向量数据库位置，修复 `faster-whisper` 多机兼容性，支持 TTS 符号自动过滤。
