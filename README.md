# 👁️🗨️ CareSight-Agent (慧眼导医): 面向视障群体的多模态医疗预问诊智能体

## 📖 项目简介 (Project Introduction)
CareSight-Agent 是一个致力于“科技向善”的开源多智能体医疗辅助系统。本项目专为视障人士、老年人及阅读困难群体设计，旨在打破医疗信息获取的视觉与专业壁垒。

结合计算机视觉感知（PyTorch）与大语言模型的高级逻辑编排（LangGraph），本系统能够通过语音和图像接收用户的模糊病症描述或药品/化验单信息，并利用动态的多智能体工作流进行分析。系统外挂了目前最大规模的中文医疗问答数据集 Huatuo-26M，采用 RAG（检索增强生成）技术，确保为弱势群体提供的每一条医疗建议都具备权威事实支撑，坚决杜绝医疗“幻觉”。

## ✨ 核心特性 (Key Features)

* **♿ 视障优先的多模态交互 (Voice Interaction)**: 系统已原生支持语音录制与播报。
  * **ASR**: 使用 `faster-whisper` (C++ 重写版) 实现毫秒级语音转文字。
  * **TTS**: 使用 `edge-tts` (微软神经语音) 实现高保真、带感情的医疗播报，并自动过滤 Markdown 特殊符号。
* **🤖 基于 LangGraph 的多智能体协作流 (Multi-Agent Orchestration)**: 摒弃传统的单次对话模型，采用 LangGraph 图状态机编排复杂的医疗推理。
* **📚 权威的百万级医疗知识底座 (RAG Engine)**: 
  * **数据源**: 集成 Huatuo-26M 专业医疗百科。
  * **向量库**: ChromaDB (持久化于 `data/chroma_db`)。
  * **Embedding**: 本地 Ollama (`embedding-gemma`) 或 API 模式。

## 🛠️ 技术架构 (Tech Stack)

* **AI 编排**: LangChain, LangGraph
* **语音模型**: `faster-whisper` (Base/Small 模型可选)
* **语音合成**: `edge-tts` (Edge Neural Voices)
* **大语言模型 (LLM)**: Google Gemini (Gemini-1.5-Flash / Gemini-3-Flash)
* **向量数据库**: ChromaDB
* **依赖库**: `sounddevice`, `pygame-ce`, `soundfile`

## 💖 社会愿景 (Social Impact)

我们相信，最好的技术应当服务于最需要它的人。CareSight-Agent 不仅仅是一段代码，它是通向医疗平权的一座桥梁，让看不见世界的人，也能被世界温柔以待。

## 💡 为什么选择 CareSight-Agent？

这是一个非常深刻且直击痛点的问题。常规的医疗问诊系统（例如普通的医疗问答网页或 App）大多是“文本输入 - 文本输出”的单模态被动交互，它们在设计时默认了用户具备正常的视觉阅读能力、屏幕打字能力，以及对环境的视觉感知能力。

对于视障群体（盲人及低视力人群，BVI）而言，这种常规系统是充满壁垒的。我们正在构思的 CareSight-Agent 并非简单的“医疗百科字典”，而是一个主动的、多模态的（语音+视觉）、且具有极高安全性约束的智能辅助系统。

具体来说，我们的项目与常规医疗问诊有以下四大核心区别，这也是它能为视障用户带来的巨大实质性帮助：

### 1. 打破“物理世界”的信息壁垒：多模态化验单与药品解析
* **常规系统**：需要用户自己看懂纸质化验单，手动将“白细胞偏高”、“血糖值 8.0”等数据打字输入给 AI。
* **对视障用户的帮助**：视障人群在面对排版复杂、充满生僻医学术语的纸质体检报告、化验单或药品说明书时，几乎无法独立获取信息。我们的系统可以利用视觉大语言模型（VLMs）和 OCR 技术充当用户的“眼睛”。结合 LangGraph，系统能够自动提取化验单上的数值，对照医学参考范围进行结构化分析，最后将冰冷的数字转化为通俗易懂的语音摘要和饮食建议播放给用户。

### 2. 极高容错率的“感知-推理-行动”主动循环
* **常规系统**：当用户输入的信息模糊，或上传的照片不清晰时，常规大模型通常会“不懂装懂”地给出通用回复。
* **对视障用户的帮助**：视障用户在拍摄药盒或描述症状时，由于视力受限，很容易拍糊或者描述不准。在 LangGraph 的多智能体框架下，我们可以构建“感知-行动循环（Perception-Action Loops）”。如果智能体发现视觉输入质量太差，它不会瞎猜，而是会主动通过语音引导用户（例如：“您拍摄的药盒有些模糊，请把手机稍微拿远一点，或者让我为您打开闪光灯”），通过这种动态的多轮语音追问，弥补视障用户在信息输入阶段的困难。

### 3. 应对视障群体“AI 幻觉”的高脆弱性（安全性保障）
* **常规系统**：大语言模型经常会产生“幻觉”（即生成看似合理但错误的信息）。普通人如果看到 AI 说错了一种药，还可以自己去核去核对说明书。
* **对视障用户的帮助**：视障用户往往高度依赖且无法轻易通过视觉去二次核对 AI 给出的信息，如果系统误读了药物剂量，后果将是致命的。因此，我们的项目引入了 LangGraph，通过多智能体的机制来降低风险：例如设置一个专门的“验证智能体（Verification Agent）”，将识别到的药物信息与权威医疗数据库强制交叉比对。这种结合了 RAG 检索和确定性规则验证的架构，能为视障用户提供比普通 AI 更高标准的安全保障。

### 4. 结合您 PyTorch 优势的“物理空间导航”潜力
* **常规系统**：仅限于线上文本交流，脱离物理空间。
* **对视障用户的帮助**：就医过程不仅仅是问诊，还包括在复杂的医院环境中找路、找科室，或者在家里寻找特定颜色的药瓶。您可以将之前开发的 PyTorch 高精感知算法融入进来，使系统具备“主动目标搜索”能力，结合手机摄像头和触觉（振动）及语音反馈，实时引导视障用户的双手去触摸或定位特定的医疗物品。

**总结来说**：常规医疗问诊系统是在解决**“我不知道生了什么病”**的问题；而 CareSight-Agent 解决的是**“我看不见我的病历、我找不到我的药、我无法用键盘打字查询，请你做我的医学智囊和眼睛”**的问题。
